# Chess Example Integration

The chess app is an example game that demonstrates how a browser game can attach
to Game Network without moving all game logic into the networking library.

The implementation lives in:

```text
single-file-chess-game/index.html
```

It is intentionally single-file because it is easy to distribute and inspect.
The important network sections are commented in the file.

## Standalone Mode

Open the page without network parameters:

```text
/single-file-chess-game/
```

or from a local checkout:

```text
single-file-chess-game/index.html
```

No Game Network code runs in this mode. It is just the local chess app.

## Direct Developer Mode

Direct developer mode is not standalone. Before opening a URL with
`transport=websocket`, a Game Network service or WebSocket hub must already be
running.

For ordinary local development, non-privileged service mode is enough:

```bash
npm run service
```

The service prints the reachable host IP. Use that printed address in direct
developer URLs. For example, if the service prints
`http://192.168.0.197:9201`, open:

Host:

```text
http://192.168.0.197:9201/single-file-chess-game/index.html?transport=websocket&role=host&room=CHESS-1&peer=chess-host&hostColor=white&signaling=ws://192.168.0.197:9201/ws
```

Guest:

```text
http://192.168.0.197:9201/single-file-chess-game/index.html?transport=websocket&role=guest&room=CHESS-1&peer=chess-guest&host=chess-host&signaling=ws://192.168.0.197:9201/ws
```

For restricted networks use the privileged/single-port service:

```bash
npm run service -- --port 80
```

That binds port `80` and uses `/ws` for signaling. Then open:

```text
http://192.168.0.197/single-file-chess-game/index.html?transport=websocket&role=host&room=CHESS-1&peer=chess-host&hostColor=white&signaling=ws://192.168.0.197/ws
http://192.168.0.197/single-file-chess-game/index.html?transport=websocket&role=guest&room=CHESS-1&peer=chess-guest&host=chess-host&signaling=ws://192.168.0.197/ws
```

Normal players should not use these direct URLs. They use `/host/:room` and
`/join/:room` generated by the launcher.

Avoid `file://...?...transport=websocket` for networked mode. ES module imports
and browser security rules are much more predictable when the game is served by
the Game Network HTTP service.

## Integration Points

The chess app exposes three small browser globals.

### `window.__CHESS_NETWORK_DIAGNOSTICS__`

Updates the `Collegamento` panel:

```js
window.__CHESS_NETWORK_DIAGNOSTICS__.update({
  status: "connected",
  connectedPeerId: "chess-guest",
  rttMs: 6,
  bestRttMs: 4,
  offsetMs: 123.4,
  lastSyncAt: Date.now(),
});
```

This is UI-only. It is useful for debugging but not required by the room API.

### `window.__CHESS_GAME_ADAPTER__`

This is the game/network boundary:

- `getSnapshot()` returns the host-authoritative chess state;
- `applySnapshot(snapshot)` applies a host snapshot on guests;
- `applyMove(move)` validates and applies guest moves on the host;
- `setNetworkRole(role, localColor, options)` configures role, color, and board
  perspective.

For another game, create an equivalent adapter with that game's snapshot and
input format.

### `window.__CHESS_GAME_NETWORK__`

This is created by the network bootstrap after the room starts:

- host exposes `sendSnapshot()`;
- guest exposes `sendLocalMove(move)`;
- guest exposes `sendRestartRequest()`;
- both keep references to the room, transport, and clock responder for debug.

## Host Flow

1. Parse URL parameters.
2. Create `WebSocketTransport`.
3. Create `ClockSyncHost`.
4. Create `HostRoom`.
5. Assign colors.
6. On guest join, send an initial snapshot.
7. On guest move input, apply it through the chess adapter.
8. Broadcast the new snapshot.

## Guest Flow

1. Parse URL parameters.
2. Create `WebSocketTransport`.
3. Create `ClockSyncHost` so the host can also sample this peer if desired.
4. Create `GuestRoom`.
5. Join the host room.
6. Read assigned player/color from `join_accept`.
7. Start `ClockSyncClient` against the host.
8. Apply host snapshots as the source of truth.
9. Send local moves as compact input messages.

## What To Copy For Another Game

Do copy:

- a small adapter boundary;
- URL/bootstrap parsing;
- host room setup;
- guest room setup;
- snapshot/input distinction;
- diagnostics panel pattern if useful.

Do not copy blindly:

- chess FEN handling;
- color assignment rules;
- promotion logic;
- chess-specific optimistic move behavior.
